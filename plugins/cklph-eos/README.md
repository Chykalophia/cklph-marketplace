# cklph-eos

Peter's operating-cadence discipline as a Claude plugin — the **quarterly priorities · weekly sync · scorecard · issues-list · people-fit · accountability** rhythm. Inspired by the operating-system-for-a-business approach popularized by various business-OS frameworks, **codified here in Peter's voice with no branded language**.

## What this is

Operating discipline for a growing team / business. **Not a republishing of any branded business-OS framework** — trademarked names belong to their owners. This plugin captures the *operating discipline* in Peter's voice.

## When this earns its keep

- You have (or are adding) team members.
- Quarterly priorities are unclear or slip routinely.
- Weekly syncs are status-update theater, not problem-solving.
- Issues recur because they get "discussed" but never resolved.
- People-fit problems linger because there's no shared language to name them.

For a true solo operator with no team, much of this is overkill — start with `cklph-os` core (`triage` / `decide` / `daily-frame` / `weekly-review`).

## What's inside (v0.1.0)

- `rocks` — quarterly priorities: how to pick 3-7, define "done", surface misses early.
- `weekly-sync` — the structured team meeting cadence (issues → discuss → solve, not status theater).
- `scorecard` — weekly leading-metrics discipline; scoreboard vs dashboard.
- `issues-ids` — Identify → Discuss → Solve loop. Resolve the root issue, not the symptom of the week.
- `people-fit` — get-it / want-it / capacity check; how to read the actual mismatch.
- `accountability-map` — clarify who owns what by role, not title.

## Install

```bash
claude plugin install cklph-eos@cklph
```
