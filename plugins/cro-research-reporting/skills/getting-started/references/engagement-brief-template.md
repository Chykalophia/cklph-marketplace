# Engagement Brief — template & conventions

The shared context the whole toolkit reads first. Keep it short, current, and specific.

## Where it lives
Default path: **`./cro-engagement.md`** in the user's working folder. If the user prefers another location, record that and use it. One brief per client/engagement.

## How skills use it
Every skill checks for this file at the start of a task and tailors to it (client, product, tier, ICP, in-scope surface, goal/metric, voice). The `getting-started` skill / `/kickoff` command create and update it. Confirm before overwriting.

## Template (copy into ./cro-engagement.md)

```markdown
# Engagement Brief — <Client/Brand>
_Last updated: <date> · Owner: <name>_

## Business
- Brand / what they sell:
- Price tier: mass | premium | luxury  (typical price / AOV: )
- Channels / platform: (e.g., Shopify; store URL; backend domain)

## Audience (ICP)
- Who buys:
- What they value (in priority order):
- What they're skeptical about:

## Scope & goal
- Surface(s) in scope: PDP | collection | cart | checkout | landing | lead-gen | SaaS signup | full funnel
- Primary goal & metric: (e.g., add-to-cart rate, revenue-per-visitor, lead conversion)
- Secondary / guardrail metrics: (e.g., return rate, AOV)
- Known baseline numbers: (pull from analytics if available)

## Voice & brand
- Tone: (e.g., confident, warm, non-hype)
- Banned words / rules: (e.g., no em dashes, no "leverage/synergy")
- Brand tokens: colors / fonts / logo / button style (if known)

## Data & access
- Connected: analytics/Shopify MCP? web search? theme/codebase access?
- Deliverable audience: internal-technical | client-facing

## Constraints & notes
- Timeline / deadlines:
- Off-limits / dependencies:
- Open questions:
```

## Update cadence
- Refresh when scope, goal, or baseline numbers change.
- Keep it to one page; move detail into the actual deliverables.
- If a number is unverified, mark it so (the research and audit skills will tier it).

## Why this exists
Research on the best skill suites (e.g., coreyhaines31/marketingskills) shows a shared, foundational context file is what makes a set of skills behave coherently rather than as disconnected tools. This brief is that layer.
