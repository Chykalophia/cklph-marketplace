# Parallel Research Brief Template

When fanning out research into parallel streams (subagents or sequential passes), give each one a precise, self-contained brief. Vague briefs return shallow, unsourced results.

## Template

```
You are a [domain] researcher. Produce a deep, source-cited brief on [stream topic],
for [audience/purpose]. Depth matters — do many searches and fetch primary sources.

CONTEXT: [2–5 sentences on the subject so the research is tailored — product, price,
audience, the problem, what "good" looks like.]

RESEARCH MANDATE — cover ALL of these with DATA and SOURCES:
1. [specific sub-question]
2. [specific sub-question]
... (8–12 concrete sub-questions; be exhaustive about the angle)

OUTPUT FORMAT: A well-organized markdown brief. For EVERY statistic, include the source
name + full URL + publication year inline. Provide concrete examples with URLs. After each
major section add a one-line "APPLY TO [subject]:" takeaway. Prioritize authoritative,
recent (2023–2026) sources [name the best ones for this domain]. Do NOT invent statistics
or URLs — if a figure is approximate or unverifiable, say so explicitly.
```

## Design rules
- **Independence:** streams must not overlap, so parallel work isn't redundant and sections don't repeat.
- **Specificity:** 8–12 concrete sub-questions per stream beats a broad theme — it forces depth and coverage.
- **Provenance baked in:** the output format demands URLs + years and forbids fabrication, every time.
- **"Apply to X" takeaways:** make each finding actionable for the specific subject, not generic.
- **Name the good sources:** tell the stream which authorities to prefer for that domain.

## Worked example — streams for a CRO/PDP report
1. **PDP/funnel best practices & benchmarks** — conversion benchmarks by tier/device, Baymard PDP findings, above-fold, sticky ATC, imagery, variant UX, speed/CWV. Sources: Baymard, NN/g, Portent, Google, Statista, Shopify.
2. **Premium buyer psychology & positioning** — luxury decision drivers, price justification, storytelling, sustainability sequencing, naming, gifting. Sources: McKinsey, Bain, Deloitte, First Insight, NielsenIQ, HBR, academic.
3. **Best-in-class teardowns** — fetch live PDPs of named comparables + modular/"platform + add-ons" analogs; for each: URL, element/section order, imagery, social proof, configurator UX, "what to steal."
4. **Specific levers** — reviews/UGC/video/fit/AR/urgency/trust/BNPL/OOS with quantified lift + examples. Sources: Spiegel, PowerReviews, Bazaarvoice, Fittingbox, FTC, vendor benchmarks (labeled).

## After the streams return
- De-duplicate and reconcile (streams may surface the same stat — keep the most primary).
- Run the verification pass on load-bearing claims.
- Map each stream to a section of the report; carry the citations through.
