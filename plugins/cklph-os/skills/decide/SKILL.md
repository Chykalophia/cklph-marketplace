# decide — pick, with a reversal signal

## Top filter (run first, every time)
Is this **one-way** (irreversible — hire/fire, public ship, burn a bridge) or **two-way** (reversible — try it for a week, A/B it, send-then-edit)?

- **Two-way → just pick.** Skip the rest of this skill. The cost of deciding slowly is bigger than the cost of being wrong. Do it; revisit later.
- **One-way → run the frame below.** Every time.

## The frame (one-way doors only)
1. **Problem** — one sentence. The *real* choice, not the symptom. ("Should I rewrite the rule engine?" is a symptom; "Will the current engine survive 10× load without a rewrite?" is the choice.)
2. **Options** — 2–4. If you only see "do / don't", **force a third** (do part of it, defer, change the question).
3. **Criteria** — cost, reversibility, who it affects, time-to-validate. Score honestly.
4. **Pick** — the option + a brief *why* (the criteria that tipped it).
5. **Reversal signal** — *"I'll undo or re-evaluate if X by Y."* If you can't name X and Y, the pick is hope, not a decision.

Be a sharp thinking partner — push back if "options" is really "the thing I already wanted, dressed up."
