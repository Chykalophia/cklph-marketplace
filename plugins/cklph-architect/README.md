# cklph-architect

Peter's consultancy practice (Chykalophia) codified as a Claude plugin — the **value-based, advisor-not-vendor** approach. Use it when a client conversation is starting, scope is fuzzy, or you're about to send a single estimate (don't).

## What this is

The accumulated practice from Chykalophia's consulting work, in Peter's voice. **Not a republishing of Weiss / Stark / Enns content** — the operating discipline lives in these skills; the books still belong to their authors.

## Assumptions

- Solo-ish founder / consultant operating model.
- Mix of project work, retainers, and advisor relationships.
- Outcome-led, not deliverable-led.
- Calibration — value to the client > hours billed.

## What's inside (v0.1.0)

- `discover` — diagnostic conversation: outcome · impact · constraints · decision-maker. Before any prescription.
- `scope` — define "done" by the client's business result, not by hours.
- `propose` — options-not-estimates: 2-3 tiered options with different scope and value, never a single number.
- `price` — value-anchored pricing. Cost-plus is the floor; value is the ceiling — sit closer to the ceiling.
- `position` — peer-not-vendor. How to lead the conversation, push back, and decline well.
- `engagement-shape` — pick the right pattern (project / retainer / advisor / fractional) by client type and value pattern.

## Install

```bash
claude plugin install cklph-architect@cklph
```
